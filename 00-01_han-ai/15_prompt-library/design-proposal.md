# プロンプトライブラリ管理ツール 設計案（たたき台）

作成日：2026-07-02　／　状態：**方針未確定・実装未着手**（この文書は意思決定用のたたき台）

## 1. このツールの目的（確定済みの前提）

- 業種別・用途別のプロンプト集を一元管理する
- クライアントへの「お土産」として渡せる形にする（研修成果物・オーダーメイド納品物・伴走先への提供素材）
- 研修の「スライドのプロンプト例は [ ] を書き換えるだけで使えます」という約束を、コース横断で支える台帳になる

## 2. 実装方式の比較

| 観点 | A. Electronアプリ | B. Notion | C. GitHub Pages 静的Web（推奨） |
|---|---|---|---|
| 作る手間 | 大（UI・保存・エクスポート全部自作） | 小（DBテンプレを作るだけ） | 中（診断ツールと同じHTML/CSS/JSスタック） |
| 自分の管理のしやすさ | ○ ローカル完結 | ◎ どこでも編集 | ◎ Git管理・Claude Codeで一括編集できる |
| クライアントへの渡しやすさ | △ PDF書き出しを実装する必要 | ○ 共有リンク（ただしNotionアカウント誘導が発生） | ◎ URLを渡すだけ／印刷CSSでPDF化も容易 |
| 業種・用途での絞り込み | 自作 | ◎ DBフィルタ標準装備 | ○ タグフィルタを実装（理解度テストで実装経験あり） |
| 納品物の見た目統一（ざつね屋ブランド） | ○ | △ Notionの見た目に固定 | ◎ 既存ダークテーマ（#1C1C1C/#DA7756）を流用 |
| 費用 | 無料 | 無料（共有相手もアカウント不要の公開リンク可） | 無料 |
| 既存資産との連携 | △ | △ | ◎ 診断ツール（proj13）の generate-library・研修資料と同居 |

## 3. 推奨案：C. GitHub Pages 静的Web ＋ マスタはJSON1ファイル

**理由：**
1. マスタデータ（prompts.json）を Git 管理すれば、価格表・研修資料と同じく Claude Code で一括メンテできる（整合性チェックの対象にできる）
2. クライアントには URL を渡すだけ。仮公開の体制（GitHub Pages）とも本公開（Netlify）とも両立する
3. 診断ツール・理解度テストで同じスタックの実装経験が2回あり、開発リスクが最も低い
4. 「業種フィルタ済みの印刷ページ」を用意すれば、オーダーメイド納品時のPDFお土産が数クリックで作れる

**Notionを併用しない理由：** マスタが二重管理になり、価格・サービス名の整合性チェックの穴になるため。

## 4. データ構造案（prompts.json）

```json
{
  "id": "sales-followup-01",
  "title": "顧客リストから個別フォローメールを一括生成",
  "prompt": "（コピーしたデータ）このリストの各社に向けた個別のフォローアップメールを作成してください。最終接触日をふまえて、自然な書き出しにしてください。",
  "placeholders": ["リスト（会社名・担当者・最終接触日）"],
  "task": "営業・顧客フォロー",
  "industries": ["共通"],
  "tools": ["Gemini", "Claude", "ChatGPT"],
  "source": "研修コース②",
  "level": "そのまま使える"
}
```

- `task`（用途）と `industries`（業種）の2軸フィルタが核
- `source` で研修・診断・伴走のどこ由来かを記録（事例蓄積を兼ねる）

## 5. 画面構成たたき台（1ページ完結）

```
┌──────────────────────────────────────┐
│ ざつね屋 プロンプトライブラリ                        │
│ [用途: ▼全て] [業種: ▼全て] [ツール: ▼全て] [検索 __] │
├──────────────────────────────────────┤
│ ┌ カード ──────────────────────────┐   │
│ │ 用途：営業・顧客フォロー   ツール：Gemini他     │   │
│ │ 顧客リストから個別フォローメールを一括生成         │   │
│ │ ┌ プロンプト本文（[ ]置き換え式） ┐ [コピー]     │   │
│ └──────────────────────────────┘   │
│ （カードが用途別に並ぶ・クリックでコピー）              │
├──────────────────────────────────────┤
│ [🖨 この絞り込み結果を印刷用に表示（納品PDF用）]          │
└──────────────────────────────────────┘
```

- バッジには必ず prefix（「用途：」「業種：」「ツール：」）を付ける（開発ルールブック準拠）
- コピーボタンは alert ではなくボタン内テキストの一時変化（「コピーしました ✓」）で通知
